# v1.2.0 design — full-fidelity client rendering

Sources: two decompile investigations (2026-07-04) of player-animation and
death/game-over/HUD systems; audio report pending. User directive: implement every
known gap, verify autonomously, then ONE human test.

## Root causes being fixed

1. **HP pushed through the raw `Health` auto-property setter** — skips
   `OnHealthChanged()` / `OnHealthChangedEvent`, so the HUD never updates and the death
   cascade (`OnStatsDeath` → `OnPlayerDeathEvent` → `DeathEvent` → `Level.playerIsDead`)
   never runs client-side. Fix: call the public `PlayerStatsManager.SetHealth(int)`.
2. **Player-animation suppression strategy**: layer-0 scrubbing can't render shooting
   (layers 1/2 weighted per-frame by the suppressed `LevelPlayerAnimationController.Update`),
   fights the game's `localScale.x` facing flip (walk-left bug), and never runs hit/flash
   (local damage events don't fire on client). Fix: **local-driven animation** — run the
   game's own controller on the client, force its polled inputs, synthesize its edge events.
3. **HUD absent**: `LevelHUD.LevelInit()` (called once from `Level.Start()`) NREs when
   force-join hasn't completed → `levelHudTemplate` stays null and nothing can recover.
   Fix: ensure join-before-`Level.Start()`, plus a guarded re-invoke fallback.
4. **Game-over/win/ghost never mirrored**: all hang off events that never fire client-side.

## Wire protocol v13

`PlayerSnapshot` additions (per player, ~3 bytes):
- `Flags` byte: bit0 Grounded, bit1 Locked, bit2 Dashing, bit3 IsShooting,
  bit4 IsUsingSuperOrEx, bit5 Invulnerable/Flashing, bit6 DashDirSign(+1/-1 as 1/0).
- `Pulses` byte (host sets bits for edges since last snapshot, client consumes once):
  bit0 WeaponFired, bit1 DamageTaken.
- LookY/MoveY come from the already-streamed input axes (v10 mirror) — no new fields.

`StateSnapshot` addition:
- `LevelFlags` byte: bit0 LevelWon (latched while the win sequence runs; host sets from a
  Harmony postfix on the win path), bit1 LevelReload (pulse when host calls
  `SceneLoader.ReloadLevel()` — same-name reload is invisible to name-diff SceneSync).

`Protocol.Version = 13`.

Host capture: flags polled off `LevelPlayerMotor`/`ArcadePlayerMotor` + `weaponManager`
each snapshot; pulses accumulated via Harmony postfixes (`OnWeaponFire` event or
`WeaponManager` fire path; `PlayerDamageReceiver.TakeDamage` postfix host-side) and
cleared after each snapshot send.

## Client changes

### A. Local-driven player animation (replaces scrubbing for players)
- Remove for players: `LevelPlayerAnimationController.Update` prefix-skip,
  `AnimatorParamPatches` blocks, layer-0 `AnimUtil.Scrub` call in `ScenePuppetry.ApplyTo`
  (players only — entities/projectiles keep scrubbing), and the facing/localScale write.
- Keep: `LevelPlayerMotor.FixedUpdate` prefix-skip; interpolated position write in
  `ScenePuppetry.ClientApply` (LateUpdate +32000).
- Extend `PlayerMotorBypass` forced state (per frame, from mirrored axes + streamed flags):
  `LookDirection`, `TrueLookDirection`, `MoveDirection`, `Grounded`, `Locked`,
  `Dashing` (+`DashDirection` sign), `IsUsingSuperOrEx`; set `weaponManager.IsShooting`
  (public setter). Hold values between snapshots (input mirror already 30 Hz-held).
- Synthesize edges: WeaponFired pulse → `LevelPlayerAnimationController.OnShotFired()`;
  DamageTaken pulse → play `Hit.Hit_Ground`/`Hit_Air` + set `hitAnimation` (reflection);
  Invulnerable flag → force `damageReceiver.state = Invulnerable` for the window so
  `flash_cr` blinks.
- Arcade levels: same approach; `ArcadePlayerAnimationController` is single-layer +
  manual animator pump; its Update runs locally too (remove its skip as well).
- Chalice/EX/super/parry long tail: explicitly OUT of v1.2.0 scope except where the
  flags already cover it (`IsUsingSuperOrEx`); noted as known limitation.

### B. HP/HUD
- Swap the reflection Health push → `PlayerStatsManager.SetHealth(int)`.
- Join-ordering: on client, force-join P1+P2 at session establish (before any level
  load); TestHarness already joins before AutoLoadLevel on host. Fallback: after a level
  scene loads, if `LevelHUD.Current` exists but has no template (reflection check),
  re-invoke `LevelHUD.LevelInit()` once when both `GetPlayer(...)` are non-null.

### C. Death / ghost / game-over / win
- Derive per-player death edges client-side from streamed IsDead transitions.
- On JustDied: spawn cosmetic ghost — `PlayerDeathEffect.Create(id, input, pos, deaths,
  PlayerMode.Level, canParry:true)` via the `LevelPlayerController.deathEffect` template
  (reflection); drive position from streamed dead-player pos while present; destroy on
  revive (IsDead→false) or scene change. Replaces the current hide-renderers approach?
  NO — keep hide-renderers for the cup body; ghost is additive.
- Both dead: reflection-set `Level.playerIsDead = true` once → stock `Level.Update`
  runs `_OnLose()` → `LevelGameOverGUI` retry card (works while paused; UI time layer).
- LevelWon flag: client runs `LevelKOAnimation.Create(isMausoleum)` + `anim_cr()` once.
  NEVER call `Level.zHack_OnWin`/`LevelEnd.Win` on client (save writes).
- LevelReload pulse: client calls `SceneLoader.ReloadLevel()` (SceneSync can't see
  same-name reloads).

### D. Audio — host-authoritative streamed SFX (per audio report)
- `AudioManager` is a static event dispatcher — `Play(string)`/`PlayLoop(string)`/`Stop(string)`
  is the process-wide chokepoint. Boss/enemy SFX come from AI coroutines (dead on client;
  NOT AnimationEvents — verified), player jump/dash/hit/death from skipped code paths.
  Level BGM + announcer intro already play client-side for free (Level is an
  AbstractPausableComponent, not AbstractLevelEntity; AudioManagerComponent autoplays on
  loader complete).
- Host: Harmony postfix on `AudioManager.Play(string)` / `PlayLoop` / `Stop` → queue
  (kind byte + key string), coalesced per snapshot tick, dedupe identical keys per tick,
  cap 16/tick. Appended to StateSnapshot (SfxCount + entries) under Protocol v13.
- Client: prefix on the same three methods returns false (suppress local gameplay SFX)
  when Mode==Client && RemoteStateSequence>0 && in a synced gameplay scene — EXCEPT when
  `CoopState.ReplayingFromHost` is set. Replay: set flag, call real AudioManager method,
  clear flag. Front-end/menu SFX stay local (scope gate). BGM untouched (local autoplay).
- Kill switch: `[Sync] EnableAudioSync` default true.
- Deliberately NOT forwarded in v1.2.0: Pan/Attenuation/pitch envelopes, dynamic BGM
  (FadeBGMVolume etc. at boss phase changes) — noted as known limitation.

## Verification plan (harness, headless)
1. Anim parity: new [Debug] `DumpAnimState` — both sides log player layer-0/1/2 state
   hashes + layer weights + localScale.x sign once/sec; compare host vs client logs.
2. Shooting: client log must show layer-1 weight = 1 while host IsShooting.
3. Damage: KillP2AfterSec run → client log shows Hit play + flash window.
4. Game-over: AutoPlay both die → client log shows `LevelGameOverGUI` In (Harmony
   postfix logs it) — assert via grep.
5. HUD: after level load, reflection-check `LevelHUD.Current` built both players; log it.
6. Ghost: JustDied → client logs ghost spawn; revive → destroy.
7. Save safety: md5 %AppData%\Cuphead before/after (BlockSaves on).
8. Full UDP regression + rates (30 Hz, no exceptions).
